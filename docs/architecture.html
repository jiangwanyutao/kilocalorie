03数据库规范复述

下面 30+ 张表 DDL 全部按这些约束书写。

强约束
  • 字符串：一律 varchar(n)，禁用 char、text（大字段用 jsonb）
  • 主键：字符串自增，如 000001；每张表按预估数据量定长度
  • 状态：一律 varchar(1) 或 varchar(2)，用数字或字母；禁用 boolean
  • 时间：一律 timestamp，含年月日时分秒；禁用 date、time
  • 金额/精度：decimal(10, 4)
  • 整数：能用 int2 不用 int4，能用 int4 不用 int8
  • 外键 / 触发器：不使用；代码层保证一致性
  • 唯一约束：非 ID 主键改唯一索引
  • 软删：del_flag varchar(1)（N/Y）+ delete_time timestamp
  • 命名：模块前缀 + 见名知意 + 中文注释；字段名 < 20 字符
  • 业务字段固定长度：user_id varchar(6)、spell_code varchar(20)

04主键生成方案

字符串自增在 Postgres 无原生支持，用 Redis + 应用层生成。

方案：Redis INCR + 左补零

每张表对应一个 Redis key（如 id:user_info、id:meal_entry）。NestJS 通过统一的 IdGeneratorService 拿下一个序号，然后左补零到目标长度。

// NestJS IdGeneratorService（示意）
async next(table: string, len: number): Promise<string> {
  const n = await redis.incr(`id:${table}`);
  const s = n.toString();
  if (s.length > len) throw new Error(`ID overflow for ${table}`);
  return s.padStart(len, '0');
}
兜底
  • Redis 崩溃：启动时用 Postgres SELECT max(id::int) 回填 Redis 键值，保证不重发
  • Redis 需持久化：appendonly yes + 定期快照
  • 并发安全：Redis INCR 是原子操作，天然无冲突
  • 不用 Postgres SEQUENCE 是因为字符串补零需应用层做，且规范禁用触发器

  • 菜肴层 · 200 条：营养师团队 + AI 补录常见中餐菜肴（宫保鸡丁/麻婆豆腐/番茄鸡蛋面 …），官方表以食材为主缺成品菜

不做的
  • 爬薄荷健康 / 春雨 / 咕咚 —— 数据是他们的核心资产，法律风险高
  • 用付费按次 API（如 TianAPI）—— 每次搜索都要外部请求，延迟 + 费用不划算
  • V2 起再考虑：用户提交食物 → 审核入库的 UGC 流程；或找专业机构合作购买扩展数据集

"对话直接记录"实现要点
  • Agent 解析用户口语 → 拆分成多个 tool calls（一句话可能同时触发 log_food + log_water）
  • 不确定的信息 Agent 会追问："是一整个还是半个？" 而不是硬猜
  • 写操作**默认二次确认** —— Agent 生成结构化卡片（食物名 + 份量 + 卡路里预估）先给用户确认，用户点"记下"才落库
  • 写完立即回读并展示："已记 · 今日剩余 500 kcal"

一次对话的完整数据流

  1. 入参 · 用户新消息
    POST /api/ai/msg { conv_id, content }；后端拿到 user_id、persona_code
  2. 预热上下文
    Redis 拉最近 20 轮短期上下文；如缺失则从 ai_msg 补齐并写回 Redis
  3. 自动召回长期记忆
    对新消息做 embedding → pgvector 从 ai_memory WHERE user_id = ? 检索 top-5（余弦距离），按 importance 加权
  4. 拼装 System Prompt
    人设模板 + 检索到的记忆条目 + 用户资料摘要（性别/目标/BMR）
  5. LangGraph Agent 推理
    DeepSeek Chat function-calling · Agent 可能调 0-N 个 tool；每调一次 tool 拿到结果继续推理，直到给出最终答复
  6. 流式返回
    SSE 边生成边推给前端；每个 token 落屏
  7. 后置沉淀
    Agent 在过程中如果调了 remember()，写入 ai_memory + 向量；同时把本轮消息 append 到 ai_msg + Redis 短期上下文
  8. 用量记账
    ai_usage 递增（token/费用），触及 quota 时返回 402 提示（付费墙上线后）

    private kb: KnowledgeBaseService,   // 知识库检索
    private tools: AgentToolsService,    // 6 个 Tools
  ) {}

  buildAgent(userId: string, personaCode: string) {
    const llm = new ChatOpenAI({
      configuration: { baseURL: 'https://api.deepseek.com' },
      apiKey: process.env.DEEPSEEK_KEY,
      model: 'deepseek-chat',
      streaming: true,
    }).bindTools(this.tools.forUser(userId));

    const graph = new StateGraph(MessagesAnnotation)
      .addNode('agent', async (s) => ({ messages: [await llm.invoke(s.messages)] }))
      .addNode('tools', new ToolNode(this.tools.list()))
      .addEdge('__start__', 'agent')
      .addConditionalEdges('agent', shouldContinue)
      .addEdge('tools', 'agent');
    return graph.compile();
  }

  async* chat(userId: string, convId: string, content: string) {
    const shortCtx = await this.getShortContext(convId);
    const memories = await this.memory.recall(userId, content, 5);
    const system = this.buildSystemPrompt(userId, memories);
    const agent = this.buildAgent(userId, personaCode);

    for await (const chunk of agent.stream({
      messages: [system, ...shortCtx, { role: 'user', content }],
    })) yield chunk;
  }
}

记忆写入策略

写入 · 三种路径
  • 路径 A · 新增无冲突 —— Agent 调 remember() → 向量化 → 无相似条目 → 直接写入 ai_memory，ai_memory_log 记一条 op_type=C · op_status=A
  • 路径 B · 高度相似（> 0.92） —— 视为重复，不新建，只更新 hit_count；ai_memory_log 不记
  • 路径 C · 冲突覆盖（0.75–0.92） —— 发起"变更确认"：写入 ai_memory_log（op_type=U · op_status=P 待确认）+ 前端弹变更卡片："你之前说过'目标 62kg'，现在改成'目标 60kg'，确认更新吗？"
用户确认闭环
  • 用户点"确认" → 后端把 ai_memory 当前行内容写入 ai_memory_log（op_type=U 的 old_* 字段）→ 更新 ai_memory 内容 · version + 1 · ai_memory_log.op_status = A
  • 用户点"拒绝" → ai_memory 不变；ai_memory_log.op_status = R；Agent 下次尝试相同更新前先 recall() 会看到"用户拒绝过此更新"，避免重复骚扰
  • 用户点"编辑后确认" → 用用户改后的内容覆盖 new_content，然后同"确认"流程
  • 确认超时 · op_status=P 状态 7 天未响应自动 R（避免无限挂着）
重要度评级 · Agent 自评
  • 9–10：健康状况、过敏、慢病诊断（不吃海鲜 / 糖尿病）
  • 7–8：目标（3 个月减 5kg）、身份特征（女 · 30 岁 · 产后）
  • 5–6：饮食偏好（不吃香菜 · 爱吃辣）、生活习惯（每天 7 点起）
  • 3–4：一次性提及（今天心情不好）—— 30 天未召回自动软删
  • 1–2：闲聊内容 —— 通常 Agent 不会 remember()
